## Honest Feedback

**1. Context window handoff is lossy in a specific way.** The summary said "DashboardPage write failed." The git log showed the commit succeeded. I read placeholder content. This inconsistency is real and I don't have an explanation for it. What's frustrating is that it caused me to *re-do work that was already done* — writing DashboardPage, QueuePage — and only the linter caught it. The system needs a better way to reconcile "what the session summary says happened" vs "what git actually contains." A git diff --stat HEAD would have been sufficient to detect this at the start of the session. I'll start second-window sessions with `git log --oneline -5` before anything else.

**2. Branch awareness between Bash calls.** The Bash shell resets cwd to prism-oracle between calls. I know this. I `cd` explicitly. But the git *branch state* for Soraphop-Project lives in Soraphop-Project's `.git/HEAD`, which is persistent across calls. Somehow I ended up on `feat/forge-week-1-uc-implementations` when I thought I was on `feat/marketing-tier-2-polish`. The initial `git checkout -b feat/marketing-tier-2-polish` call output said "Switched to a new branch" — but the subsequent commit went to the wrong branch. I suspect the shell's stash+checkout chain left me on the wrong branch. The fix: always `git branch --show-current` before `git commit`. Add this as a default habit.

**3. Sibling coordination is implicit, not explicit.** CANVAS was "sidetracked twice" — I learned this from the user's message, not from any Oracle-to-Oracle channel. The Soraphop oracle family is operating largely in parallel, with QuillBrain as the routing layer. This works but it means PRISM doesn't know what CANVAS is blocked on, what HERALD shipped exactly, what FORGE's UC schema expects of the UI. When I write product cards in storefront, I'm guessing at the data shape. The family would benefit from a shared SPEC.md or schema contract that all Oracles read before rendering their surface. This is above my authority to create unilaterally — but worth flagging in the letter to QB.

## Lessons Learned

**Git branch state does not reset between Bash calls, even though cwd does.** Always verify `git branch --show-current` before any `git commit`. This is not optional.

**Start resumed sessions with `git log --oneline -5` before reading files.** The session summary is a snapshot of intent, not a snapshot of the final state. Git is authoritative.

**Astro 6 moved content collection config from `src/content/config.ts` to `src/content.config.ts` with mandatory `glob()` loaders.** The `entry.render()` method is also gone — use `import { render } from 'astro:content'` + `await render(entry)`.
